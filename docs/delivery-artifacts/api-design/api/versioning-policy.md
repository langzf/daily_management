# Versioning Policy

Traceability IDs: REQ-001, REQ-002
Status: draft
Owner: API Team
Updated: 2026-04-01

## Policy
- Major versions for breaking changes.
- Minor versions for backward-compatible additions.
- Deprecation notice at least one release cycle before removal.

## Compatibility Rules
- Preserve existing fields and semantics within major version.
- New fields must be optional by default.
- Existing error codes must remain stable.
